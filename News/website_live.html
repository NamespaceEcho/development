You have to admit that title definitely caught your attention :>. Anyway, the website is officially live! Well, most of it anyway.


To be honest, my main reason for creating the site is to link something back to the Steam page of all my projects. However, that doesn’t mean I half-assed this or just slapped something together in the hopes it passes some kind of user experience standard.


It’s not my first time creating a website. However, it’s my first time making one that actually has a working theme with a minimal amount of details, such as the underline slants and mixing dark indigo with white. I wanted the theme to be simple, but memorable, with elements that aren’t always easy to work with, and a color scheme that doesn’t require a lot of variability.

So… why a repository and not a .com?

I’m using GitHub Pages to host the site because I don’t really want to pay for a web domain and hosting. Not that I can’t afford it, but given I’m an unemployed indie-dev still living with his parents, I’m not exactly in a position where I should be spending money right now. Yeah, I know, shocking to hear a grown ass man still with his parents in this economy, right? Not. I’m not trying to be quirky either; I really thought I’d be out of the house by now, but if you want to be in the game industry and not work under a company that’s immoral, greedy, and uses AI to generate slop or uses your code to train models, you gotta go in it for yourself.


Listen to me, I’m going on a personal tangent when I should be talking about what’s actually on the site. To be fair, it’s 6:20 AM, and I’m hungry.


Anyway, here’s a list of features on the site:


  • Scrolling Animations: Instead of using GSAP.js, I decided to write my own animation timeline code for ease of use. My system is emulated to that of how animations can be written in code in Unity. You can’t deny the fact that the syntax and setup are pretty straightforward, and more at home for a programmer than a designer.
  • Parallax Effects: I added parallax effects to images. Kind of a cliche in modern design, but honestly, I like it. Turns out, it's not as easy to add for fixed-positioned objects, so I had to use my animator for that. Luckily, with the way I set it up, it was trivial. A W for images and animations :>.
  • Mobile Friendly Design: Odds are, not everyone who visits the site will be viewing on a device with a big screen size. I added styling support for smaller screens and mobile devices.
